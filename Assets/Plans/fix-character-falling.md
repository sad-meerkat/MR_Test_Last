# Project Overview
- Game Title: MR Test (XR Multiplayer)
- Issue: Characters fall through the table floor or fall forever because they have gravity enabled and are non-kinematic, combined with potential collision tunneling.

# Game Mechanics
- Characters are joystick-controlled and stay on a flat tabletop.
- Physical boundaries (walls) are generated by `TableBoundary.cs`.

# Key Asset & Context
- `ByakuyaFighter.prefab`, `SasukeFighter.prefab`, `TableCharacter.prefab`: Currently have `isKinematic = False` and `useGravity = True`.
- `FightingGameManager.cs`: Spawns fighters but doesn't parent them.
- `TableCharacterSummoner.cs`: Spawns the general character but doesn't parent it.

# Implementation Steps

## Step 1: Update Prefab Physics
- **Description**: Set `isKinematic = True` and `useGravity = False` on the `Rigidbody` of `ByakuyaFighter`, `SasukeFighter`, and `TableCharacter` prefabs. This stops gravity-induced falling and provides more stable script-based movement.
- **Assigned role**: developer
- **Dependencies**: None
- **Parallelizable**: Yes

## Step 2: Implement Parenting in Spawners
- **Description**:
    1. Modify `FightingGameManager.cs`: Parent spawned fighters to the manager's transform or a container under the table.
    2. Modify `TableCharacterSummoner.cs`: Parent the spawned character to the summoner's transform.
- **Assigned role**: developer
- **Dependencies**: None
- **Parallelizable**: Yes

## Step 3: Adjust Fighter Spawn Height
- **Description**: Raise the spawn positions in `FightingGameManager.cs` by `0.05f` (5cm) to match the logic in `TableCharacterSummoner`, ensuring characters spawn clearly above the table floor.
- **Assigned role**: developer
- **Dependencies**: None
- **Parallelizable**: Yes

# Verification & Testing
- **Spawn Test**: Summon characters and verify they appear on the table and stay at the correct height without falling.
- **Movement Test**: Use joysticks to move characters. Verify they move smoothly on the table surface.
- **Boundary Test**: Verify characters still stop when hitting the table walls (Note: Kinematic-to-static collisions require the wall to have a collider).
